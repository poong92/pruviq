## MACD가 실제로 측정하는 것

MACD(이동평균수렴확산)는 두 EMA 사이의 모멘텀 차이를 포착합니다. 핵심 계산식은 단순합니다:

```
MACD 라인   = EMA(12) - EMA(26)
신호 라인   = MACD 라인의 EMA(9)
히스토그램  = MACD 라인 - 신호 라인
```

12기간 EMA가 26기간 EMA보다 앞서 나가면 MACD가 상승합니다 — 단기 모멘텀이 장기 모멘텀을 앞서고 있다는 의미입니다. 크로스오버는 그 관계가 뒤집힐 때 발생합니다.

## 크로스오버 신호

MACD 크로스오버는 MACD 라인이 신호 라인을 교차할 때 발생합니다:

```
MACD가 신호 라인 위로 교차 → 상승 크로스오버 (LONG 진입 후보)
MACD가 신호 라인 아래로 교차 → 하락 크로스오버 (SHORT 진입 후보)
```

논리는 명확합니다: MACD 라인이 신호 라인 위로 교차하면 모멘텀이 위쪽으로 가속 중이라는 의미입니다. 히스토그램이 음수에서 양수로 전환됩니다 — 판단이 아닌 측정 가능한 명확한 이벤트입니다.

거래 가능하게 만드는 것은 **히스토그램 제로 크로스**입니다: 크로스오버 정확한 순간에 바가 방향과 색을 바꿉니다.

## 크립토에서 작동할 수 있는 이유

크립토 시장은 전통 자산보다 강하게 추세를 만듭니다. 코인이 압축 국면(예: 볼린저 밴드 스퀴즈)에서 돌파할 때, 모멘텀이 여러 시간 동안 지속될 수 있습니다. MACD 크로스오버는 다음 시나리오에서 특히 유용합니다:

- **추세 확인**: 스퀴즈 돌파 이후 상승 크로스오버는 모멘텀이 실제임을 시사
- **모멘텀 가속**: 상승하는 히스토그램 바는 움직임이 강화되고 있음을 보여줌
- **방향 필터**: MACD가 제로 아래에서 하락 교차하면, 추세 맥락이 숏 신호에 무게를 더함

약점: 횡보 시장에서는 MACD 라인과 신호 라인이 붙어 다니며 잦은 거짓 교차를 생성합니다. 이것이 크로스오버 신호 단독으로는 충분하지 않은 이유입니다.

## PRUVIQ에서 읽는 방법

PRUVIQ 시뮬레이터에서 모든 전략 결과는 신호 분석에 MACD 데이터를 포함합니다. 전략 구성에서 사용 가능한 필드:

| 필드 | 의미 |
|------|------|
| `macd` | MACD 라인 값 (EMA12 빼기 EMA26) |
| `signal` | 신호 라인 값 (MACD의 EMA9) |
| `histogram` | MACD와 신호의 차이 |
| `crossover` | 상승 교차가 발생한 캔들에서 True |
| `crossunder` | 하락 교차가 발생한 캔들에서 True |

Strategy Builder에서 필터로 사용하려면: `crossover = true`를 기본 신호(예: 스퀴즈 돌파)와 함께 조건으로 설정하세요. 이렇게 하면 MACD 모멘텀이 분석과 일치하는 거래만 진입합니다.

## MACD 크로스와 다른 신호 결합

MACD 크로스오버 단독으로는 품질이 낮은 신호가 너무 많이 발생합니다. 백테스트 데이터가 일관되게 지지하는 패턴은 **확인 레이어**로 사용하는 것입니다:

```
SHORT 진입 예시 (BB 스퀴즈 + MACD):
  1. 이전 캔들에서 BB 스퀴즈 감지
  2. BB 폭 확장 >= 10% (돌파 확인)
  3. 신호 대비 MACD 크로스언더 = true (하락 모멘텀)
  4. 거래량 비율 >= 2.0 (움직임에 실제 신뢰도)
```

PRUVIQ가 테스트한 500개 이상 코인에서, MACD 모멘텀 정렬을 요구하는 전략은 순수 가격 신호 접근 방식보다 의미 있게 낮은 위양성률을 보였습니다.
